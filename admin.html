Email Recipients

Recipient lists are stored encrypted (this is a public repository - plain text would publish members' addresses). Enter the recipients key to unlock the editor. It is remembered on this device, protected by your PIN.

First-time setup - do this once:

  1. Click Generate to create a key. Copy it somewhere safe.
  2. Add it as a repository secret named RECIPIENTS_KEY here.
  3. Come back, unlock with the key, type in the recipient lists, and Save.

Until the encrypted file exists AND the secret is set, the emails keep using the recipient lists already stored in GitHub secrets - nothing breaks in the meantime.

Update a Secret (write-only)

Write-only by design: the value is encrypted in this browser to the repository's public key and sent straight to GitHub - it is never stored, logged or displayed anywhere, and even GitHub cannot show it back. Overwriting is permanent (the old value cannot be recovered), and workflows pick the new value up on their next run. Requires the device token to have Secrets: Read and write. Each change is recorded in the Admin Actions log - name only, never the value.
