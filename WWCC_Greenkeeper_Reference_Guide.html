Overview & Data Sources

Where the data comes from and how often it updates

The dashboard draws live data from three separate sources, each updating on its own schedule. No manual data entry is required - everything refreshes automatically.

Data Sources

🌡️
Davis WeatherLink Station (Your On-Site Sensor)
Provides live current conditions every 60 seconds: temperature, humidity, wind speed/direction, rainfall today, evapotranspiration (ET), and pressure. Historical daily data is fetched hourly via the WeatherLink v2 API to build the 7-day record.
🌐
Open-Meteo (Bureau-Quality Forecast)
A high-resolution Australian weather forecast service. Provides 4-day hourly and daily forecasts every 30 minutes - used for the Next Spray Window, UV timer, and irrigation forecast. Freely available, no API key required.
💾
On-Device Memory (7-Day Store)
The browser stores a rolling 7-day record locally on the device. Each day records: ET, rainfall, max/min/mean temperature, mean humidity, and a rain-day flag. This is what powers disease models, GDD calculations, and weekly reports. Data older than 7 days is automatically deleted.

Auto-Refresh Schedule

Every 15 secAll tiles, spray status, mow window, Next Spray Window - all UI panels
Every 60 secLive station data - temperature, wind, rain, ET, pressure (Davis v1)
Every 30 min4-day forecast (Open-Meteo) and today's 24h rolling leaf wetness data (Davis v2)
Every 60 minHistorical daily weather data (Davis v2) stored to 7-day local record
Every 5 minUpdate check - detects if a new version of the dashboard is available
🔴
Data Freshness Indicator - The "last updated" timestamp at the top of the dashboard turns red if the station data is more than 10 minutes old. This means the connection to the Davis station has dropped and the displayed values may be stale.

Delta T & Wet-Bulb Temperature

The key spray conditions indicator - Stull (2011) formula

Delta T (ΔT) is the difference between dry-bulb temperature and wet-bulb temperature. It is the most reliable single indicator of spray evaporation conditions. Too low = droplets stay wet and run off or bounce; too high = droplets evaporate before reaching the target leaf.

Too Low
ΔT < 2°C
Droplets won't evaporate - runoff risk, reduced efficacy
Ideal (Contact)
ΔT 2–8°C
Contact fungicides, herbicides, insecticides
Ideal (Systemic)
ΔT 2–10°C
Systemic fungicides, growth regulators, wetting agents
Too High
ΔT > 12°C
Droplets evaporate before hitting the leaf - total loss

The Calculation - Stull (2011) Wet-Bulb Formula

The wet-bulb temperature is calculated using the Stull (2011) empirical approximation - the same method recommended by APVMA and AgVic for pesticide application decisions.

Tw = T × arctan(0.151977 × √(RH + 8.313659)) + arctan(T + RH) − arctan(RH − 1.676331) + 0.00391838 × RH1.5 × arctan(0.023101 × RH) − 4.686035 ΔT = T − Tw Where: T = dry-bulb temperature (°C) RH = relative humidity (%) Tw = wet-bulb temperature (°C)
ℹ️
The Dew Point Depression (DPD) used in the Mow tab and disease leaf wetness model uses a separate but related Magnus formula:
DPD = Air Temperature − Dew Point. The dew point is the temperature at which the air becomes fully saturated (100% RH).

Next Spray Window - 48-Hour Scan

The dashboard scans the next 48 hours of hourly forecast data from Open-Meteo. For each hour it:

  1. Calculates Delta T from forecast temperature and humidity (Stull 2011)
  2. Checks Delta T is within the selected product's ideal range
  3. Checks temperature is below the product's max
  4. Checks wind speed is below the product's max
  5. Checks rain probability is below 30%

Hours passing all four checks are Ideal. Hours where 2 of 3 primary parameters are OK and none have hard-failed are Marginal. Consecutive qualifying hours are grouped into a window. The earliest window is displayed with its start time and duration.

24-Hour Wind Forecast Strip

Shows the next 24 hours of wind speed and direction, colour-coded to the selected product's wind thresholds (green = below ideal max, yellow = warning zone, red = no-go). Scroll sideways on narrow screens to see all hours.

GDD measures heat accumulation above the minimum temperature a grass needs to grow. It tells you how fast the grass is growing and helps time growth regulator (PGR) applications.

Disease Tab

Four disease models with exact conditions and risk thresholds

All four disease models run on a rolling 24-hour window of sub-hourly data from the Davis WeatherLink v2 API (readings every 5–10 minutes), updated every 30 minutes. No physical leaf wetness sensor is required.

How "Leaf Wetness" Is Determined

The station infers leaf wetness from atmospheric conditions. An hour is counted as wet if either condition is true:

💧
Condition 1: Rain occurred during that period (any measurable rainfall > 0 mm)
OR Condition 2: Dew Point Depression ≤ 2.7°C AND wind speed ≤ 3.6 m/s

When the air is very close to saturation (small DPD) and wind is calm, dew or condensation forms on leaf surfaces. Sub-hourly records are deduplicated so each calendar hour counts only once.

Disease 1 - Dollar Spot (Sclerotinia homoeocarpa)

Warm days with cool humid nights. The fungus infects during leaf wetness periods at temperatures between 15–30°C. The dashboard uses the Smith-Kerns logistic regression model from Penn State University - one of the most validated turf disease models in the world.

Mow Tab

Dew detection and burn-off time estimation

The mow decision is based on Dew Point Depression (DPD) - the difference between current air temperature and the dew point. When DPD is small, dew or surface moisture is likely on the leaf, causing bruising, disease spread, and clipping issues if mown.

Threshold
DPD ≥ 3.5°C
Surface is dry - safe to mow
Dew Present
DPD < 3.5°C
Dew or surface moisture likely - wait

Dew Point Calculation (Magnus Formula)

gamma = ln(RH ÷ 100) + (17.625 × T) ÷ (243.04 + T) Dew Point = (243.04 × gamma) ÷ (17.625 − gamma) DPD = Air Temperature − Dew Point Where T = current temperature (°C), RH = relative humidity (%)

Burn-Off Time Estimation

When dew is present, the dashboard scans hourly forecast data forward hour by hour. It finds the first hour where:

Forecast Temperature − Current Dew Point ≥ 3.5°C

That hour is displayed as the estimated safe mowing time. This updates automatically as forecast data refreshes every 30 minutes.

Irrigation Tab

7-day water balance and clay soil moisture status

Water Balance Formula

The dashboard tracks soil moisture using a simple but effective daily deficit model. ET represents water lost from the soil; rain represents water gained.

Daily Deficit (mm) = ET Today − Rain Today 7-Day Running Balance = Σ (ET per day) − Σ (Rain per day) over 7 days Positive balance = soil is drying (ET has exceeded rainfall) Negative balance = soil has a surplus (rain has exceeded ET)
💧
ET Source: Your Davis station calculates ET using the Penman-Monteith method from temperature, humidity, solar radiation, and wind measured on site. Forecast ET for the 3-day outlook comes from Open-Meteo's FAO-56 Penman-Monteith reference ET.

Clay Soil Moisture Zones

The five zones are calibrated for Wagga Wagga's heavy clay soils. Clay holds water longer than sandy soils but is more prone to waterlogging and cracking at extremes.

Pressure Trend Algorithm

The dashboard stores the last 10 pressure readings with timestamps in a ring buffer in memory. The trend is calculated from the difference across the full buffer window:

Rate (hPa/hr) = (Newest pressure − Oldest pressure) ÷ Hours elapsed Requires at least 2 readings with ≥ 2 minutes between them

Ticker Bar & Screen Messages

Unified alert ticker + custom messages sent to the golfer page and TV kiosk

How the Ticker Bar Works

All weather alerts and custom management messages are shown in a single sticky ticker bar at the top of every page (GK, golfer, and TV kiosk). When more than one item is active, the bar cycles through them one at a time with a 0.7-second fade transition. When nothing is active, the bar hides completely.

📢
What can appear in the ticker:
⚡ Thunderstorm alert (auto - WMO code 95/96/99 in next 4 hours)
❄️ Frost alert (auto - overnight low below 4°C)
🌫️ Fog alert (auto - fog forecast tomorrow morning)
🛺 Carts on paths (auto - 7-day surplus >10mm or 24h rain ≥15mm)
💬 Custom management messages (manually posted via the 📢 panel)

The automatic weather alerts use the same trigger thresholds documented in Sections 4 and 5. They appear and disappear without any manual action.

Posting a Custom Screen Message

The 📢 button sits in the top-right corner of the GK Dashboard, next to the "Last updated" timestamp. Clicking it opens a slide-in panel from the right side of the screen.

StepAction
1. Open panelClick the 📢 button near "Last updated" (top right of the GK page)
2. Type messageEnter your message text in the text box - keep it short enough to read in the ticker
3. Set durationChoose how long the message stays active (see duration options below)
4. PostClick Post Message - the message is saved to GitHub and appears within 2 minutes on all screens
5. RemoveTap the × button next to any active message in the panel list to remove it immediately

GitHub PAT Setup (One-Time)

🔑
A GitHub Personal Access Token (PAT) is required to write messages to the shared message file.

To create one: GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens → New token.
Set Repository access to Bidgee182/wwcc-weather-page and grant Contents: Read and write.

Replace the placeholder ghp_REPLACE_WITH_YOUR_TOKEN in the GITHUB_PAT line near the top of index.html with your generated token. This only needs to be done once.
✅
If the PAT is not yet set, the 📢 panel will still open and allow you to compose a message, but the Post Message button will show an error. Weather alert items (lightning, frost, fog, carts) continue to work without a PAT - they never write to GitHub.
